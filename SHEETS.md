# 구글 시트로 내용 관리하기

시트에 적으면 사이트가 페이지를 열 때마다 그 내용을 읽어옵니다.
서버도 DB도 필요 없고, 사이트를 다시 배포할 필요도 없습니다. **시트만 고치면 끝입니다.**

---

## 1. 시트 만들기

1. [sheets.new](https://sheets.new) 로 새 스프레드시트를 만듭니다
2. 아래쪽 탭 이름을 정확히 세 개로 맞춥니다 — `명령어`, `멤버`, `현황`
3. `sheet-template` 폴더의 CSV 세 개를 각 탭에 넣습니다
   - 탭을 고른 뒤 `파일 → 가져오기 → 업로드` → 해당 CSV
   - 가져오기 위치는 **현재 시트 바꾸기** 를 고르세요

CSV를 안 쓰고 직접 만들 거라면 **첫 줄은 반드시 제목 줄**이어야 합니다. 사이트는 첫 줄을 건너뛰고 읽습니다.

### 탭별 열 순서

**명령어**

| A 명령어 | B 설명 | C 분류 | D 관리자전용 |
|---|---|---|---|
| /신입 | 신입 가이드를 시작합니다. | 신입 안내 | N |
| /쉿 | 채팅 통제를 시작합니다. | 운영 | Y |

- C열 분류가 같은 것끼리 묶여 구분선이 그어집니다. **같은 분류는 붙여서 적으세요.**
- D열에 `Y`를 적으면 "운영자 전용" 표시가 붙습니다. 비워두면 일반 명령어입니다.

**멤버**

| A 닉네임 | B 나이 | C 키 | D 전공 or 직업 | E 쉬는 요일 | F 취미 | G MBTI | H 본인의 매력 | I 이상형 | J 흡연유무 & 주량 | K 하고싶은 말 |
|---|---|---|---|---|---|---|---|---|---|---|

A열 닉네임이 비면 그 줄은 무시됩니다.

**현황**

| A 닉네임 | B 상태 | C 상대 | D 복귀 예정 | E 메모 |
|---|---|---|---|---|
| 호랑 | 외출 | | 오늘 22시 | 저녁 약속 |
| 하늘 | 매칭 | 바다 | | |

B열에 `외출`이 들어가면 노란 점, 아니면 매칭(민트 점)으로 표시됩니다.

---

## 2. 공유 설정 (이걸 안 하면 안 보입니다)

시트 오른쪽 위 **공유** → 일반 액세스를 **링크가 있는 모든 사용자** → 역할은 **뷰어**

역할은 반드시 **뷰어**로 두세요. 편집자로 두면 링크를 아는 사람이 시트를 고칠 수 있습니다.

---

## 3. 시트 ID를 사이트에 넣기

시트 주소가 이렇다면

```
https://docs.google.com/spreadsheets/d/1AbC2DeF3GhI4JkL5MnO/edit#gid=0
```

가운데 `1AbC2DeF3GhI4JkL5MnO` 가 ID입니다.

`assets/config.js`

```js
SHEET_ID: "1AbC2DeF3GhI4JkL5MnO",

SHEETS: {
  commands: "명령어",
  members: "멤버",
  status: "현황"
}
```

```bash
git add assets/config.js
git commit -m "구글 시트 연동"
git push
```

---

## 4. 확인

사이트를 열어 시트 내용이 나오는지 봅니다.
시트에서 한 줄 고치고 사이트를 새로고침하면 바로 바뀌어 있어야 합니다.

---

## 시트를 쓰면 달라지는 것

| | 시트 안 씀 | 시트 씀 |
|---|---|---|
| 원본 위치 | 각자 브라우저 | 구글 시트 하나 |
| 다른 사람에게 보이나 | 안 보임 | **보임** |
| 수정 방법 | 사이트의 추가·수정 버튼 | 시트에 직접 입력 |
| 여러 명이 관리 | 불가능 | 가능 (시트 편집 권한만 주면 됨) |
| 배포 다시 필요 | — | 필요 없음 |

시트를 켜면 사이트의 추가·수정·삭제 버튼은 **사라집니다.** 원본이 두 군데면 반드시 어긋나기 때문입니다. 관리자 링크로 들어가면 그 자리에 **시트에서 수정하기** 버튼이 대신 나옵니다.

운영진을 늘리려면 시트 공유에서 그 사람을 **편집자**로 추가하면 됩니다. 사이트 관리자 키를 나눠줄 필요가 없어집니다.

---

## 알아둘 점

**반영 속도.** 보통 새로고침하면 바로 반영됩니다. 가끔 구글이 잠깐 이전 내용을 돌려줄 때가 있는데, 30초쯤 뒤 다시 새로고침하면 됩니다.

**쉼표와 줄바꿈.** 셀 안에 쉼표나 따옴표가 있어도 괜찮습니다. 셀 안에서 줄을 바꾸는 것(`Alt+Enter`)도 그대로 나옵니다.

**시트에서 사이트로만 흐릅니다.** 사이트에서 고친 내용이 시트로 돌아가지는 않습니다. 그래서 시트를 켜면 사이트 수정 버튼을 없앤 것입니다.

**개인정보.** 시트는 "링크가 있는 모든 사용자"에게 열려 있습니다. 주소를 아는 사람은 시트 원본을 볼 수 있으니, 방 사람들에게 보여줄 내용만 넣으세요. 연락처나 실명 같은 건 넣지 마시기를 권합니다.

---

## 안 될 때

| 증상 | 원인 |
|---|---|
| 아무것도 안 나오고 "시트에 연결하지 못했습니다" | 공유가 **링크가 있는 모든 사용자**가 아님 |
| "○○ 탭을 찾지 못했습니다" | 탭 이름과 `config.js`의 이름이 다름. 띄어쓰기까지 같아야 합니다 |
| 첫 줄 데이터가 안 보임 | 첫 줄은 제목 줄로 건너뜁니다. 데이터는 2행부터 |
| 분류가 뒤죽박죽 섞임 | 같은 분류끼리 줄을 붙여 놓으세요 |
| 고쳤는데 그대로 | 30초 뒤 `Ctrl+Shift+R` 로 강제 새로고침 |
| 수정 버튼이 안 보임 | 정상입니다. 시트 모드에서는 시트에서 고칩니다 |

---

## 시트를 그만 쓰고 싶으면

`config.js`의 `SHEET_ID` 를 다시 빈 값으로 두면 됩니다. 사이트에서 직접 수정하는 원래 방식으로 돌아갑니다.
