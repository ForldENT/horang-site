# 호랑봇 안내소 사용법

이 사이트를 어떻게 쓰는지 정리한 문서입니다.
**앞부분(1~2장)은 방 사람들에게 그대로 보여줘도 되는 내용**이고, 3장부터는 운영자용입니다.

---

# 1장. 이 사이트는 무엇인가

카카오톡 오픈채팅방에서 자주 묻는 것들을 채팅 위로 올려가며 찾지 않아도 되게 만든 페이지입니다.

네 가지가 있습니다.

| 화면 | 여기서 알 수 있는 것 |
|---|---|
| **명령어** | 봇에게 뭐라고 쳐야 하는지, 그러면 무슨 일이 생기는지 |
| **멤버 소개** | 방 사람들의 닉네임, 나이, 취미, 이상형 등 자기소개 |
| **매칭 · 외출** | 지금 누가 매칭 중이고 누가 자리를 비웠는지 |
| **패치노트** | 봇과 방 규칙이 언제 어떻게 바뀌었는지 |

맨 위 메뉴로 오갑니다. 휴대폰에서도 그대로 됩니다.

---

# 2장. 방 사람들이 쓰는 법

## 명령어 찾기

- **검색창**에 단어를 치면 바로 걸러집니다. `로또`만 쳐도 관련 명령어가 다 나옵니다
- **분류 선택**으로 `신입 안내`, `운영`, `생활 · 재미` 처럼 묶어 볼 수 있습니다
- 왼쪽이 채팅에 **그대로 치는 말**, 오른쪽이 그때 봇이 하는 일입니다
- `운영자 전용` 표시가 붙은 건 방장·부방장만 쓸 수 있습니다

## 멤버 소개 보기

- 검색창에 닉네임, 취미, MBTI 아무거나 쳐도 걸립니다. `ENFP` 로 검색하면 그 MBTI인 사람만 나옵니다
- 자기 소개를 올리거나 고치고 싶으면 **운영자에게 말해주세요.** 사이트에서 직접 쓸 수는 없습니다

## 매칭 · 외출 확인

- **민트색 점**은 매칭 중, **노란 점**은 외출 중입니다
- 위쪽에서 매칭만 / 외출만 골라 볼 수 있습니다

## 패치노트 읽기

최근 것이 맨 위입니다. 분류로 걸러 볼 수 있습니다.

- **신규 기능 추가** (민트) — 새로 생긴 명령어나 기능
- **오류 수정** (노랑) — 안 되던 게 고쳐진 것
- **기존 기능 삭제** (회색) — 없어진 것

## 자주 묻는 것

**수정 버튼이 안 보여요** — 정상입니다. 방 사람들에게는 보기 전용 링크가 갑니다.

**내용이 틀렸어요** — 방에서 운영자에게 알려주세요.

**즐겨찾기 해도 되나요** — 네. 휴대폰이라면 홈 화면에 추가해두면 앱처럼 열립니다.

# 3장. 운영자가 쓰는 법

## 두 개의 링크

| 대상 | 주소 | 할 수 있는 것 |
|---|---|---|
| 방 사람들 | `.../horang-site/` | 보기만 |
| 운영자 | `.../horang-site/?key=관리자키` | 추가 · 수정 · 삭제 |

- 관리자 링크는 **개인 메시지로만** 보내세요. 단톡방에 올리면 누구나 수정할 수 있게 됩니다
- 관리자 상태는 **탭을 닫으면 풀립니다**. 홈의 `나가기` 버튼으로도 풀 수 있습니다
- 방에 공유할 주소는 홈의 **일반 공유 주소 복사** 버튼으로 복사하면 실수가 없습니다

키가 새어나간 것 같으면 `assets/config.js` 의 `ADMIN_KEY` 를 바꾸고 push하세요. 기존 관리자 링크는 즉시 무효가 됩니다.

## 내용 고치는 방법 — 두 가지 중 하나

지금 어느 방식인지는 관리자로 홈에 들어가면 안내 문구로 알 수 있습니다.

### 방식 A — 구글 시트 (여러 명이 관리할 때)

시트에 적으면 사이트가 열릴 때마다 그 내용을 읽어옵니다. 누가 열든 같은 내용이 보입니다.

- 수정은 **시트에서** 합니다. 사이트의 추가·수정 버튼은 사라지고 `시트에서 수정하기` 버튼이 대신 나옵니다
- 시트를 고치고 사이트를 새로고침하면 바로 반영됩니다. 다시 배포할 필요 없습니다
- 운영자를 늘리려면 시트 공유에서 **편집자**로 추가하면 됩니다. 관리자 키를 나눠줄 필요가 없습니다

설정 방법은 [SHEETS.md](SHEETS.md) 참고.

### 방식 B — 사이트에서 직접 (혼자 관리할 때)

관리자 링크로 들어가 각 화면의 추가·수정·삭제 버튼을 씁니다.

**주의:** 이 방식은 **내가 쓰는 브라우저에만 저장됩니다.** 다른 사람에게도, 내 휴대폰에도 안 보입니다. 기기를 옮길 때는 홈의 `데이터 내보내기` → 다른 기기에서 `데이터 가져오기` 를 쓰세요.

여러 명이 관리하거나 방 사람들에게 실제로 보여주려면 **방식 A를 쓰셔야 합니다.**

## 화면별로 관리하는 것

### 명령어

봇 명령어가 늘거나 없어질 때마다 여기를 맞춰주세요.

- **분류**를 같은 이름으로 맞추면 자동으로 묶입니다. 시트에서는 같은 분류끼리 줄을 붙여 놓으세요
- **관리자전용**을 켜면 `운영자 전용` 표시가 붙습니다. 방 사람들에게도 보이긴 하지만, 자기가 쓸 수 없다는 걸 알게 됩니다

### 멤버 소개

새 인원이 들어와 자소서를 쓰면 여기에 옮겨 적습니다. 11개 항목 중 **닉네임만 필수**이고 나머지는 비워도 그 줄이 안 나올 뿐 문제없습니다.

방을 나간 사람은 삭제해 주세요. 남아 있으면 신입이 착각합니다.

### 매칭 · 외출

가장 자주 바뀌는 화면입니다.

- **상태**는 `매칭` 또는 `외출` 둘 중 하나
- 매칭이면 **상대** 닉네임을 적습니다
- 외출이면 **복귀 예정**에 `오늘 22시` 처럼 적습니다
- 끝난 매칭이나 돌아온 사람은 지워주세요. 안 지우면 이 화면을 아무도 안 믿게 됩니다

### 패치노트

봇이나 방 규칙이 바뀔 때마다 남깁니다.

| 항목 | 필수 | 설명 |
|---|---|---|
| 패치 날짜 | 필수 | 오늘 날짜가 기본값으로 잡힙니다 |
| 패치 분류 | 필수 | 신규 기능 추가 / 오류 수정 / 기존 기능 삭제 |
| 버전 | 선택 | `1.2.0` 처럼. 안 쓰면 비워두세요 |
| 패치 내용 | 필수 | 줄바꿈도 그대로 보입니다 |

**내용은 방 사람 입장에서 적으세요.** "핸들러 리팩터링" 말고 "`/눈치게임` 이 중간에 멈추던 문제를 고쳤습니다" 처럼요.

버전은 안 써도 됩니다. 쓸 거라면 규칙만 정해두세요. 흔한 방식은 이렇습니다.

- 기능이 새로 생기면 가운데 숫자 올리기 (1.2.0 → 1.3.0)
- 오류만 고쳤으면 끝자리 올리기 (1.2.0 → 1.2.1)

# 4장. 디자인 바꾸기

색과 글꼴은 `assets/style.css` 맨 위 `:root` 에 모여 있습니다. `theme-lab.html` 을 열면 색을 눈으로 고르고 CSS를 복사해 갈 수 있습니다.

자세한 건 [DESIGN.md](DESIGN.md) 참고.

# 5장. 문제가 생겼을 때

| 증상 | 해결 |
|---|---|
| 흰 배경에 파란 링크만 나옴 | `assets` 폴더가 HTML과 같은 위치에 없음 |
| 화면이 비어 있음 | 시트 공유 설정이 `링크가 있는 모든 사용자` 인지 확인 |
| 고쳤는데 반영이 안 됨 | 30초 뒤 `Ctrl+Shift+R` 로 강제 새로고침 |
| 수정 버튼이 안 보임 | 일반 링크로 들어왔거나, 시트 모드임 |
| 수정한 게 다른 기기에서 안 보임 | 방식 B(브라우저 저장)를 쓰는 중. 시트로 옮기세요 |
