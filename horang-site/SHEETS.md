# 구글 시트 연결하기

두 가지 방식이 있습니다. 원하시는 건 **B**입니다.

| | A. 읽기 전용 | B. 읽기 + 쓰기 |
|---|---|---|
| 시트에 적으면 사이트에 나옴 | O | O |
| **사이트에서 적으면 시트에 저장됨** | X | **O** |
| 준비물 | 시트 공유 설정만 | 시트에 Apps Script 붙이기 (10분) |
| 비용 | 0원 | 0원 |

A만 먼저 해두고 나중에 B로 올려도 됩니다. B를 하면 A 설정은 필요 없습니다.

---

# 0. 공통 — 탭 네 개

지금 시트(`호랑봇 자소서`)에 탭이 네 개 있어야 합니다.

| 탭 이름 | 담는 내용 |
|---|---|
| `명령어` | 봇 명령어 |
| `자소서` | 멤버 소개 |
| `매칭외출` | 매칭 · 외출 현황 |
| `패치노트` | 패치노트 |

탭 이름은 바꿔도 되지만, 바꾸면 `assets/config.js` 의 `SHEETS` 와 Apps Script 의 `TABS` 를 같이 고쳐야 합니다.

**B 방식으로 가실 거면 탭을 직접 안 만들어도 됩니다.** 스크립트가 자동으로 만들어 줍니다. B의 1번으로 바로 가세요.

직접 만든다면 `sheet-template` 폴더의 CSV 네 개를 각 탭에 넣으면 열 순서까지 맞춰집니다.
`파일 → 가져오기 → 업로드` → 가져오기 위치는 **현재 시트 바꾸기**.

**첫 줄은 반드시 제목 줄**이어야 합니다. 사이트는 첫 줄을 건너뛰고 읽습니다.

## 탭별 열 순서

**명령어**

| A 명령어 | B 설명 | C 분류 | D 관리자전용 |
|---|---|---|---|
| /신입 | 신입 가이드를 시작합니다. | 신입 안내 | N |
| /쉿 | 채팅 통제를 시작합니다. | 운영 | Y |

C열 분류가 같은 것끼리 묶여 구분선이 그어집니다. 같은 분류는 붙여서 적으세요.
D열에 `Y` 면 "운영자 전용" 표시가 붙습니다.

**자소서**

| A 닉네임 | B 나이 | C 키 | D 전공 or 직업 | E 쉬는 요일 | F 취미 | G MBTI | H 본인의 매력 | I 이상형 | J 흡연유무 & 주량 | K 하고싶은 말 |
|---|---|---|---|---|---|---|---|---|---|---|

A열 닉네임이 비면 그 줄은 무시됩니다.

**매칭외출**

| A 닉네임 | B 상태 | C 상대 | D 복귀 예정 | E 메모 |
|---|---|---|---|---|
| 호랑 | 외출 | | 오늘 22시 | 저녁 약속 |

B열에 `외출` 이 들어가면 노란 점, 아니면 매칭(민트 점)입니다.

**패치노트**

| A 날짜 | B 분류 | C 버전 | D 내용 |
|---|---|---|---|
| 2026-07-20 | 신규 기능 추가 | 1.0.0 | 안내소를 열었습니다. |

A열과 D열은 필수. 날짜는 `2026-07-20` `2026.7.20` `2026년 7월 20일` 다 알아봅니다.
분류는 세 가지 중 하나. 순서는 아무렇게나 적어도 사이트가 최신순으로 정렬합니다.

# A. 읽기 전용으로 쓰기

## 1. 공유 설정

시트 오른쪽 위 **공유** → 일반 액세스를 **링크가 있는 모든 사용자** → 역할은 **뷰어**

역할은 반드시 뷰어로 두세요. 편집자로 두면 링크를 아는 사람이 시트를 고칠 수 있습니다.

## 2. 설정 넣기

`assets/config.js` — 시트 ID는 이미 넣어뒀습니다.

```js
SHEET_ID: "1jiInOSutI4f5mVDwJ_cxAyWcfsmKGhT-lfJLdOYtF10",
SCRIPT_URL: "",   // 비워두면 읽기 전용
```

```bash
git add assets/config.js
git commit -m "구글 시트 연결"
git push
```

끝입니다. 이 방식에서는 사이트의 추가·수정 버튼이 사라지고, 관리자에게는 `시트에서 수정하기` 버튼이 대신 나옵니다.

# B. 사이트에서 적으면 시트에 저장되게 하기

## 1. 스크립트 붙이기

1. 시트를 열고 **확장 프로그램 → Apps Script**
2. 왼쪽에 `Code.gs` 가 있습니다. 안에 있던 내용을 **전부 지우고**,
   `apps-script/Code.gs` 파일 내용을 **전부 붙여넣기**
3. 맨 위 `ADMIN_KEY` 를 바꿉니다. 사이트 `config.js` 의 `ADMIN_KEY` 와 **똑같아야 합니다**

```js
const ADMIN_KEY = "여기에_긴_임의의_문자열";
```

4. 저장 (`Ctrl+S`)

## 2. 권한 승인 + 탭 만들기

1. 편집기 위쪽 함수 목록에서 **setup** 을 고르고 **실행**
2. "권한 검토" 창 → 본인 계정 선택
3. **"Google에서 확인하지 않은 앱"** 경고가 나옵니다. 내가 만든 스크립트이니 정상입니다.
   **고급 → (프로젝트 이름)(으)로 이동 → 허용**
4. **승인이 끝나면 실행 버튼을 한 번 더 누르세요.** 승인만 하고 끝나는 경우가 많습니다
5. 편집기 아래 **실행 로그** 에 이렇게 나오면 성공입니다

```
연결된 시트: 호랑봇 자소서
새로 만든 탭: 명령어, 자소서, 매칭외출, 패치노트
--- 여기까지 보이면 성공입니다. 시트를 새로고침하세요. ---
```

6. 시트 탭으로 돌아가 **새로고침(F5)** 하면 탭 네 개가 보입니다

### 탭이 안 생길 때

실행 로그를 보면 원인이 나옵니다.

| 로그 내용 | 원인과 해결 |
|---|---|
| 로그가 아예 비어 있음 | 실행이 안 된 것. 함수 목록이 `setup` 인지 확인하고 다시 실행 |
| `시트를 찾지 못했습니다` | 스크립트가 시트에 연결돼 있지 않음. 아래 참고 |
| `SHEET_ID 로 시트를 열지 못했습니다` | ID가 틀렸거나, 실행한 구글 계정에 그 시트 권한이 없음 |
| `연결된 시트: ...` 가 나오는데 탭이 안 보임 | 시트 화면을 새로고침(F5) 하세요 |
| `이미 있던 탭: ...` 만 나옴 | 이미 만들어져 있는 것. 정상입니다 |

**스크립트가 시트에 연결됐는지 확인하는 법**
편집기 왼쪽 아래 **개요** 를 누르면 "컨테이너" 항목에 시트 이름이 나옵니다.
안 나온다면 `script.google.com` 에서 따로 만든 프로젝트입니다. 이 경우 코드 위쪽

```js
const SHEET_ID = "1jiInOSutI4f5mVDwJ_cxAyWcfsmKGhT-lfJLdOYtF10";
```

가 채워져 있으면 그대로 동작합니다. 비어 있다면 채우고 다시 실행하세요.

**가장 확실한 방법**은 시트에서 **확장 프로그램 → Apps Script** 로 새로 여는 것입니다. 그러면 처음부터 시트에 붙은 상태로 만들어집니다.

## 3. 웹 앱으로 배포

1. 오른쪽 위 **배포 → 새 배포**
2. 톱니바퀴 → **웹 앱** 선택
3. 설정
   - 설명: 아무거나
   - **다음 사용자로 실행: 나**
   - **액세스 권한이 있는 사용자: 모든 사용자** ← 이게 핵심입니다
4. **배포** → 나온 **웹 앱 URL** 복사

주소는 이렇게 생겼습니다.

```
https://script.google.com/macros/s/AKfycb..................../exec
```

`/exec` 로 끝나야 합니다. `/dev` 는 본인만 쓸 수 있는 주소라 안 됩니다.

## 4. 사이트에 연결

`assets/config.js`

```js
ADMIN_KEY: "스크립트에 넣은 것과 똑같은 값",
SHEET_ID: "1jiInOSutI4f5mVDwJ_cxAyWcfsmKGhT-lfJLdOYtF10",
SCRIPT_URL: "https://script.google.com/macros/s/AKfycb..../exec",
```

```bash
git add assets/config.js
git commit -m "Apps Script 연결"
git push
```

## 5. 확인

1. 관리자 링크로 들어가 아무 화면에서나 하나 추가
2. **시트를 새로고침** → 방금 적은 게 들어와 있으면 성공
3. 시크릿 창에서 일반 링크를 열어 같은 내용이 보이는지 확인

---

## 스크립트를 고친 다음에는

Apps Script 는 **저장만으로는 반영되지 않습니다.**
**배포 → 배포 관리 → 연필(수정) → 버전을 "새 버전" → 배포** 를 해야 합니다.

이렇게 하면 주소가 그대로 유지됩니다. "새 배포"를 누르면 주소가 새로 생기니 주의하세요.

---

## 이 방식이 어떻게 동작하나

```
[사이트]  ──읽기(GET)──▶  [Apps Script]  ──▶  [구글 시트]
          ──저장(POST)─▶   관리자 키 확인
```

- **읽기**는 누구나 됩니다. 방 사람들이 페이지를 열 때마다 시트 내용을 받아옵니다
- **저장**은 관리자 키가 맞을 때만 됩니다. 키는 관리자 링크(`?key=...`)로 들어올 때만 브라우저에 잠깐 저장되고, 탭을 닫으면 사라집니다
- 저장은 **해당 탭 전체를 다시 쓰는 방식**입니다. 두 사람이 동시에 저장해도 섞이지 않도록 잠금을 걸어뒀습니다

## 알아둘 점

**시트를 직접 고쳐도 됩니다.** 사이트에서 쓰든 시트에서 쓰든 결과는 같습니다. 다만 사이트를 열어둔 채로 시트를 고치면, 사이트가 열릴 때 받아온 옛 내용을 기준으로 저장해서 시트 수정분이 덮일 수 있습니다. **시트를 직접 고쳤으면 사이트를 새로고침하고 나서 작업하세요.**

**서식은 유지됩니다.** 스크립트는 값만 지우고 다시 씁니다. 셀 색이나 글꼴은 그대로 남습니다. 다만 데이터 줄 수가 줄면 아래쪽 서식이 빈 칸에 남아 보일 수 있습니다.

**개인정보.** A 방식이면 시트가 링크로 열려 있어 주소를 아는 사람은 원본을 볼 수 있습니다. B 방식이면 시트를 비공개로 둘 수 있어 더 안전합니다. 어느 쪽이든 자소서에 연락처나 실명은 넣지 마시길 권합니다.

---

## 안 될 때

| 증상 | 원인 |
|---|---|
| 화면이 비고 "Apps Script 주소와 배포 권한을 확인해 주세요" | 액세스 권한이 **모든 사용자** 가 아님. 또는 주소가 `/dev` 로 끝남 |
| 저장하면 "관리자 키가 맞지 않습니다" | `config.js` 의 `ADMIN_KEY` ≠ `Code.gs` 의 `ADMIN_KEY` |
| 저장은 되는 것 같은데 시트에 없음 | 스크립트를 고친 뒤 재배포를 안 함 |
| "○○ 탭을 찾지 못했습니다" (A 방식) | 탭 이름과 `config.js` 의 이름이 다름. 띄어쓰기까지 같아야 합니다 |
| 첫 줄 데이터가 안 보임 | 첫 줄은 제목 줄로 건너뜁니다. 데이터는 2행부터 |
| 고쳤는데 그대로 | 30초 뒤 `Ctrl+Shift+R` 로 강제 새로고침 |
| 수정 버튼이 안 보임 | A 방식(읽기 전용)이거나 일반 링크로 들어옴 |

---

## 시트를 그만 쓰고 싶으면

`config.js` 의 `SHEET_ID` 와 `SCRIPT_URL` 을 모두 빈 값으로 두면 됩니다. 브라우저에만 저장하는 원래 방식으로 돌아갑니다.
